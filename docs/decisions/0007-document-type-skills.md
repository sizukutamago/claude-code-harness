# 0007: ドキュメント種別ごとのスキル化

**Status:** 検討中
**Date:** 2026-04-05

## 背景

現在、ドキュメントの配置ルール（docs-structure.md）はあるが、各ドキュメント種別（ADR、設計書、調査資料等）を「いつ・どう書くか」のスキルがない。ADR は配置ルールだけ定義されていて、書くタイミング・テンプレート・フローが未定義。

## アイデア

ドキュメント種別ごとにスキルを作る:

- **adr** — 設計判断が発生したタイミングで ADR を記録するスキル（テンプレート、記録トリガー、既存 ADR との整合確認）
- **design-doc** — 設計書を書くスキル（brainstorming の出力を設計書フォーマットに構造化）
- **research-doc** — 調査結果を記録するスキル（調査→比較→結論の構造化）

## 検討ポイント

- brainstorming スキルの design.md 出力と design-doc スキルは重複しないか？
- ADR は brainstorming の副産物（選択肢の中から1つを選んだ記録）として自動生成すべきか、独立スキルにすべきか？
- スキルが増えすぎると認知負荷が上がる。ユーティリティスキルとして必要なものだけ作るべき

## 次のアクション

- brainstorming スキルの出力と ADR の関係を整理してから判断する
