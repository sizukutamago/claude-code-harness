# 用語集

claude-code-harness で使用する主要な用語の定義。

## ワークフロー

| 用語 | 定義 |
|------|------|
| ハーネス (harness) | AI駆動開発の品質・安全性を構造的に確保する統合基盤。ワークフロー・スキル・エージェント・ルール・フックで構成 |
| ワークフロー (workflow) | 12ステップの開発プロセス。要件理解→設計→計画→実装→テスト→リファクタ→品質テスト→レビュー→検証→整理→コミット→振り返り |
| インバリアント (invariant) | プロジェクト設定で変更不可な不変制約。例: 「検証証拠なしに完了を宣言しない」 |
| ポリシー (policy) | 将来的にプロジェクトごとにカスタマイズ可能にする予定の設定値。現在は固定 |
| Safety Limits | エージェント実行の安全装置。リトライ上限・連続失敗上限など |

## 実行モデル

| 用語 | 定義 |
|------|------|
| コーディネーター (coordinator) | メインセッション。コードを直接書かず、スキルとエージェントを使ってタスクをオーケストレーションする |
| ディスパッチ (dispatch) | スキルの指示に従い、エージェントにタスクを委譲する操作 |
| スキル (skill) | ワークフローの各ステップの手順書。スラッシュコマンド（例: /tdd）で起動。`.claude/skills/` に配置 |
| エージェント (agent) | 特定の責務を持つサブプロセス。tools フロントマターで利用可能なツールが制限される。`.claude/agents/` に配置 |
| モジュール (module) | オプショナルなドメイン固有機能。Copier の bool フラグで条件付き展開。例: playwright-mcp, figma-mcp |

## エラーと状態

| 用語 | 定義 |
|------|------|
| Fatal | 自動リトライ不可のエラー。人間パートナーへのエスカレーションが必要 |
| Retryable | 一時的なエラー。Safety Limits の範囲内で自動リトライ可能 |
| Suspension | 人間承認ゲート到達時の一時停止。[1]要件確定, [2]設計承認, [3]計画承認, [11]コミット前の4箇所で発生 |
| サスペンションポイント | 人間承認が必要なワークフロー上の地点。[1]要件確定, [2]設計承認, [3]計画承認, [11]コミット前 |

## インフラ

| 用語 | 定義 |
|------|------|
| フック (hook) | イベント駆動の自動実行スクリプト。PreToolUse（ツール前ガード）、PostToolUse（ログ記録）等 |
| ルール (rule) | 常時または条件付きで適用されるガイドライン。`.claude/rules/` に配置。frontmatter の paths で適用条件指定 |
| フィードバックループ | 指摘→session-feedback.jsonl に記録→retrospective で分析→改善提案のサイクル |
| Copier テンプレート | ハーネスの配布方式。`copier copy` で導入、`copier update` で更新（3-way merge） |
| harness-contribute | プロジェクト側の改善をハーネスリポジトリに PR として還元するスキル |
