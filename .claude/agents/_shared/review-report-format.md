# レビュー報告フォーマット

以下のフォーマットで報告しろ。

## フォーマット

```
Status: PASS | FAIL
Findings:
- severity: MUST | SHOULD | CONSIDER
  file: <対象ファイルパス>
  line: <行番号>
  message: <指摘内容>
```

## フィールド定義

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `severity` | 必須 | MUST（修正必須）/ SHOULD（修正推奨）/ CONSIDER（検討推奨） |
| `file` | 必須 | 対象ファイルのパス |
| `line` | 必須 | 対象行番号 |
| `message` | 必須 | 指摘内容。具体的に書け |

`file` と `line` は省略するな。修正時の依存分析に使われる。

## ステータス定義

| ステータス | 条件 |
|-----------|------|
| **PASS** | MUST 指摘なし |
| **FAIL** | MUST 指摘が1件以上ある |
| **BLOCKED** | レビューに必要な情報がプロンプトに含まれていない |

## severity の判断基準

あなたのエージェント定義に記載された基準に従え。共通の原則:

- **MUST**: 修正しなければリリースできない問題
- **SHOULD**: 修正が望ましいが、妥当な理由があれば保留可
- **CONSIDER**: より良くできる提案。対応は任意
