# Security Rules

## 原則

**安全でないコードは動くコードではない。**

## 必須ルール

1. **入力バリデーション**: 外部からの入力（ユーザー入力、API リクエスト、ファイル読み込み）は信頼しない。型・形式・範囲を検証してから使う
2. **パラメータ化クエリ**: SQL やデータベースクエリにユーザー入力を直接埋め込まない。プレースホルダー・パラメータ化クエリを使う
3. **出力エスケープ**: HTML、URL、シェルコマンドに値を埋め込む際は適切にエスケープする
4. **最小権限**: ファイル、API、データベース操作は必要最小限の権限で行う
5. **エラー情報の制限**: エラーレスポンスに内部実装の詳細（スタックトレース、DB スキーマ、ファイルパス）を露出しない

## 禁止事項

- シークレット（API キー、パスワード、トークン）をコードにハードコードする
- `eval()` や動的コード実行にユーザー入力を渡す
- HTTP で機密情報を送信する（HTTPS を使う）
- パスワードを平文で保存する
- CORS を `*` に設定して全オリジンを許可する（開発環境を除く）
- ユーザー入力をそのままファイルパスに使う（パストラバーサル）
