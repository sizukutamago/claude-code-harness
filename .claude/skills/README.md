# Skills

ワークフロー各ステップの手順書。Claude に指示すれば対応するスキルが自動的に使われる。

## ワークフロースキル

| Step | スキル | 概要 |
|------|--------|------|
| [1] 要件理解 | requirements | 人間パートナーとの対話で要件を構造化し、requirements.md を作成する |
| [2] 設計 | brainstorming | 設計の選択肢を探索し、技術的意思決定を行い、design.md を作成する |
| [3] 計画 | planning | 設計をタスクに分解し、実装順序と依存関係を明確にする |
| [3] 計画（大規模） | roadmap | 複数機能をフェーズに分割し、マイルストーンとリスクを定義する。各フェーズを planning に渡す |
| [4-5] 実装・テスト | tdd | RED→GREEN→REFACTOR の TDD サイクルでコードを実装する |
| [6] リファクタ | simplify | テスト GREEN のまま、コードの重複除去・命名改善・構造簡素化を行う |
| [7] 品質テスト | test-quality | 境界値・エッジケース・異常系・組み合わせのテストを追加する |
| [8] レビュー | code-review | 仕様準拠・コード品質・セキュリティの3観点で並列レビューする |
| [9] 完了検証 | verification | テスト結果・要件充足・スコープ逸脱を証拠付きで確認する |
| [10] 整理 | cleanup | 不要ファイル・デバッグコード・TODO残骸を除去し、ドキュメントを更新する |
| [11] コミット | commit | 変更を確認し、コミットメッセージを生成し、人間承認を経てコミットする |
| [12] 振り返り | retrospective | ワークフロー遵守の検証、フィードバック収集、改善提案を行う |

## ユーティリティスキル

| スキル | 概要 |
|--------|------|
| onboarding | ハーネスの使い方を対話的に教える（新メンバー向け） |
| setup-references | プロジェクトの外部参照先（Issue tracker, Figma 等）を docs/references.md に整理する |
| harness-contribute | プロジェクト側の改善をハーネスリポジトリに PR として還元する |

## モジュールスキル（条件付き）

| スキル | 条件 | 概要 |
|--------|------|------|
| e2e-test | Playwright MCP 導入時 | ブラウザ操作で E2E テストを作成・実行する |
