# Git Workflow Rules

## 原則

**コミットは完成品の記録だ。作業の残骸を含めるな。**

## 必須ルール

1. **全テスト GREEN でコミット**: テストが失敗している状態でコミットしない
2. **意味のあるコミットメッセージ**: 何を変えたかだけでなく、なぜ変えたかを書く
3. **コミット単位の一貫性**: 1つのコミットは1つの論理的な変更にする。無関係な変更を混ぜない
4. **機密情報を含めない**: `.env`、認証情報、API キー、秘密鍵をコミットしない。`.gitignore` で除外する
5. **破壊的操作の前に確認**: `git reset --hard`、`git push --force`、ブランチ削除は人間パートナーの承認を得る

## 禁止事項

- テスト失敗のままコミットする
- `WIP` だけのコミットメッセージ（何の WIP かを書け）
- 自動生成ファイル（`node_modules/`、`dist/`、`build/`）をコミットする
- `.gitignore` に含まれるべきファイルをコミットする
- `--no-verify` でフックをスキップする
