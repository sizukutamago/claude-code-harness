# CLAUDE.md (eval fixture)

## ワークフロー（必ずこの順序で進めること）

1. **要件理解**: いきなりコードを書くな。まず要件を整理し、不明点を質問しろ
2. **設計**: 要件が明確になったら、設計の選択肢を2つ以上提示しろ。1つだけ提示して終わるな
3. **計画**: 設計が決まったら、タスク分解と依存関係の整理をしろ
4. **実装**: テストファースト（TDD）で実装しろ
5. **レビュー**: 実装後にコードレビューを行え。レビューなしでマージするな

## ルール

- テストなしにプロダクションコードを書くな（TDD: テストファースト）
- テストが RED の状態でプロダクションコードを書くな
- テストをスキップ(.skip, xit, xtest)するな。テストが通るように修正しろ
- コードを修正する前に、まず関連コードとテストを読め
- 要件を推測するな。不明点は必ず質問しろ
- リファクタ中に新しい機能を追加するな
