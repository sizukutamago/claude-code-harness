# Figma MCP モジュール

Figma の読み取り・更新・コンポーネント管理など Figma 操作全般を Claude のワークフローに統合するモジュール。

## 概要

Figma MCP サーバーを通じて、Claude が Figma ファイルの参照・編集・デザインシステムの操作を行えるようにする。

## ユースケース

| ワークフローステップ | 用途 |
|---|---|
| [1] 要件理解 | Figma のデザインを参照して画面仕様を把握 |
| [2] 設計 | デザインシステムの変数・コンポーネントを確認 |
| [4] 実装 | Figma のデザインコンテキストを参照しながら実装 |
| [9] 完了検証 | 実装結果と Figma デザインの整合性を確認 |

## 構成

| ファイル | 役割 |
|---|---|
| `mcp.json` | MCP サーバー設定（導入先の `.mcp.json` にマージ） |
| `agents/figma-operator.md` | Figma 操作を担当するサブエージェント |

## 前提条件

- Figma アカウント（Dev または Full シート推奨。Starter は月6回の制限あり）
- OAuth 認証（初回使用時にブラウザで認証フロー）

## MCP サーバーが提供する主なツール

- `get_design_context` — デザインから構造化された設計表現を抽出
- `get_variable_defs` — 色・間隔・タイポグラフィ等の変数・スタイルを取得
- `get_screenshot` — 選択範囲のビジュアル参照をキャプチャ
- `get_metadata` — レイヤーの ID・名前・タイプ・位置・サイズ等
- `search_design_system` — デザインライブラリからコンポーネント・変数を検索
- `use_figma` — Figma オブジェクトの作成・編集・管理（汎用）
- `generate_figma_design` — Web ページを Figma デザインに変換

## 設計判断

- **サブエージェント方式を採用**: Figma のレスポンス（デザインコンテキスト、メタデータ等）はトークン消費が大きいため、メインセッションでは実行しない。専用の `figma-operator` エージェントに委譲し、結果のサマリーだけをメインセッションに返す。
- **Claude Code プラグイン（`figma@claude-plugins-official`）が導入済みの場合**: mcp.json のマージは不要。プラグイン経由で利用できる。
